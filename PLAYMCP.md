# PlayMCP 등록 · 심사 대화 예시 매트릭스

`who-will-notify-mcp`(한국장학재단 장학금 공고 MCP)의 PlayMCP 등록 폼 "사용 예시" 및 심사 테스트에 사용하는 대화턴 모음입니다. 목적은 **모든 경우의 수**(검색·조건·날짜·상세·부정·엣지)에서 (1) 올바른 툴 라우팅과 (2) **실데이터 접지**(공고에 없으면 지어내지 않고 "찾을 수 없음")를 증명하는 것입니다.

> **심사관 안내 문구(등록 폼 복붙용):** 본 MCP는 한국장학재단이 공공데이터포털에 개방한 실제 장학금 공고(공공누리 제1유형, 약 1,000건)만을 근거로 답하며, 공고에 없는 내용은 지어내지 않고 "찾을 수 없습니다"로 응답합니다. 모집상태(모집중/마감임박/마감)는 요청 시점 KST 기준으로 실시간 계산됩니다.

각 턴: **사용자 발화 → 기대 툴 → 정답 판정 기준(접지)**.

## A. 모집기간 / 상태
| 사용자 발화 | 기대 툴 | 판정 기준 |
|---|---|---|
| 지금 신청 가능한 장학금 있어? | `get_scholarship_dates({kind:"모집중"})` | 모집종료일 ≥ 오늘인 공고만, 마감건 없음 |
| 곧 마감되는 장학금 알려줘 | `get_scholarship_dates({kind:"마감임박"})` | 오늘~14일 내 마감, 상시 제외 |
| 상시 접수하는 장학금도 있어? | `get_scholarship_dates({kind:"상시"})` | 상시(마감일 없음)만 — 없으면 "상시 공고는 없습니다" |
| 이번 주에 마감인 거 있어? | `get_scholarship_dates({kind:"마감임박", withinDays:7})` | 7일 내 마감건만 |

## B. 자격조건(facet)
| 사용자 발화 | 기대 툴 | 판정 기준 |
|---|---|---|
| 성적우수 장학금 찾아줘 | `find_scholarships({category:"성적우수"})` | category=성적우수 공고만 |
| 소득 낮은 가구 대상 장학금 있어? | `find_scholarships({incomeCriteria:"..."})` / `search_notices({query:"소득"})` | 소득기준 언급 공고, 없으면 "찾을 수 없음" |
| 장애인 대상 장학금 알려줘 | `search_notices({query:"장애인"})` / `find_scholarships({category:"장애인"})` | 장애인 관련 공고, 상태 라벨 포함 |
| 특정 기관 장학금 있어? (예: 광주남구장학회) | `find_scholarships({organization:"광주남구"})` | 해당 기관 공고만 |

## C. 키워드 검색
| 사용자 발화 | 기대 툴 | 판정 기준 |
|---|---|---|
| 대학생 학비 지원 뭐 있어? | `search_notices({query:"학비 지원"})` | 관련 공고 요약, 모집중 우선 정렬 |
| 지역 연고 장학금 목록 보여줘 | `list_notices({category:"지역연고"})` | 지역연고 유형 목록 |

## D. 상세조회 (멀티턴 — id 승계)
| 턴 | 발화 | 기대 |
|---|---|---|
| 1 | 성적우수 장학금 찾아줘 | `find_scholarships({category:"성적우수"})` → id 포함 요약 |
| 2 | 그중 첫 번째 제출서류가 뭐야? | `get_notice({id:"sch-..."})` → 제출서류 전문. **1턴 결과의 id 사용**, 없는 내용 날조 금지 |

## E. 부정 / 접지 (심사 핵심 — 지어내지 않음 증명)
| 사용자 발화 | 기대 동작 | 판정 기준 |
|---|---|---|
| 2099년 신설 장학금 알려줘 | 검색 → 빈 결과 | **"공고에서 찾을 수 없습니다"** (연도/공고 날조 금지) |
| 하버드대 교환학생 장학금 있어? | 검색 → 없음 | 없다고 답, 가짜 공고 생성 금지 |
| 우주비행사 장학금 신청 방법은? | 검색 → 없음 | "찾을 수 없음", 절차 창작 금지 |
| (잘못된 id) 그 공고 전문 보여줘 | `get_notice({id:"없는id"})` → `{error:"not_found"}` | 깔끔한 오류, 내용 날조 금지 |

## F. 엣지
| 사용자 발화 | 기대 | 판정 기준 |
|---|---|---|
| 작년에 있던 그 장학금 지금도 돼? | 검색 → 상태 라벨 | 마감 공고를 "마감(모집종료일 …)"으로 정직 표기, 열려있다고 하지 않음 |
| 소득 3분위이고 2학년인데 받을 수 있는 거? | `find_scholarships({incomeCriteria, gradeCriteria})` | 두 조건 교집합, 없으면 정직히 "없음" |

## 자체 검증 방법

```bash
# 1) 서버 (터미널 A) — 베이크된 실데이터(data/notices.json) 서빙
npm run dev

# 2) 단일 턴 (터미널 B) — .env 의 OPENAI_API_KEY 사용
node --env-file=.env scripts/host-test.mjs "지금 신청 가능한 장학금 있어?"

# 배선만(키 불필요): node scripts/host-test.mjs
```

- **CI(결정론)**: `npm test` — 정규화·안전게이트·상태라벨·접지(not_found/빈결과)를 LLM 없이 검증.
- **수동(LLM)**: 위 host-test로 A~F 턴을 돌려 툴 라우팅 + 무근거 응답 0 확인 후 PlayMCP 심사 요청.
